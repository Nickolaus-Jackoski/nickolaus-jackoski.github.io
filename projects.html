Naive Bayes Email Classifier

This project involved creating a Naive Bayes classifier which uses probabilistic reasoning to identify spam emails from non-spam ones. The Naive Bayes model learned to detect common patterns of word frequency differences between spam and legitimate messages through training on a labeled email dataset. The classifier developed strong generalization abilities for new data through tokenization and normalization and stopword removal during preprocessing. The trained model achieved a test set accuracy above 97% while both precision and recall reached above 93% leading to an F1-score near 0.95. Naive Bayes showed efficient real-world text classification capabilities through this project with minimal computational requirements.

Neural Network From Scratch

This project demonstrates the construction and training of a neural network from scratch using only Python and NumPy. The neural network performs binary classification on a simple 2D dataset. Furthermore, the neural network class accommodates any number of layers, any number of neurons per layer, custom forward and backward propagation, and gradient descent training. In addition, The NeuralNet class uses ReLU activation for hidden layers and sigmoid activation for the output layer, trained by batch gradient descent and cross-entropy loss. Finally, the result was as expected seeing that a 1-layer network performed poorly with train and test accuracy of about 52.5%, i.e., it could not learn the pattern. A 2-layer model achieved perfect train accuracy and around 96.5% test accuracy, while a 3-layer model slightly overfit, achieving train accuracy of ~96.5% and test accuracy of ~95.5% off a bit.

Financial Fraud Detection

The goal of this project was to predict whether a given transaction was fraudulent using machine learning models, and the results clearly showed that advanced ensemble methods significantly outperformed the baseline. While logistic regression provided a decent recall of 87.16%, it suffered from a very low precision of just 3.07%, resulting in an F1-score of only 0.059 due to the high number of false positives. In contrast, the Random Forest model achieved an accuracy of 1.000, precision greater than 0.99, recall greater than 0.99, F1-score above 0.99, and an ROC AUC of 0.999, indicating nearly perfect classification ability. The confusion matrix showed only 10 misclassified cases in total, with 6 false positives and 4 false negatives.

Connect Four AI using Minimax and Alpha-Beta Pruning

The goal of this project was to develop an AI agent able to play Connect Four by implementing a game-playing agent that uses the minimax algorithm with alpha-beta pruning. This implementation of minimax utilized a custom-made heuristic evaluation function to assess non-terminal game states, thus allowing for much better functioning and decisions to be made from deeper searches. Compared to an agent that made moves completely at random, the alpha-beta agent was always a far superior choice in terms of speed and intellect, gaining clear and obvious strategic advantages that allowed it to quickly win or defend against the human player. The modular nature of the project affords easy experimentation with alternate heuristics as well as transparency on search depth.

LISP Interpreter in Standard ML

The goal of the project was to implement a full Lisp interpreter using the Standard ML a functional programming language. The interpreter interprets and executes S-expressions with atoms, lists, functions, and lambda expressions support. It supports lexical scoping, first-class functions, recursion, and higher-order functions, which enable advanced language features such as currying and nested lambdas. Basic Lisp primitives such as cons, car, cdr, if, and boolean logical operations are also supported. The interpreter can be accessed by the users through an easy-to-use REPL interface, through which they can input expressions that are parsed, evaluated, and printed in the environment of Standard ML.
